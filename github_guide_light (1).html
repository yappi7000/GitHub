1
GitHubのアカウントを作る
最初に1回だけやれば OK
ブラウザで https://github.com を開いて「Sign up」をクリック。
メールアドレス・パスワード・ユーザー名 を入力するだけです。
💡 ユーザー名は英語と数字だけにする
ユーザー名が「yappi7000」なら、公開URLは
https://yappi7000.github.io/〜/ という形になります。
日本語は使えません。英語と数字とハイフンだけにしてください。
✅ ログイン成功の確認
ログインすると「Dashboard」という画面が表示されます。
左上に「Dashboard」と書いてあれば成功です。
2
リポジトリを作る
「リポジトリ」＝ファイルを入れるフォルダのようなもの
ダッシュボード画面の右上に 「＋」ボタン があります。
それをクリックしてメニューを開きます。
▼ 画面イメージ：右上の「＋」をクリックするとこのメニューが出る
＋ ▾

1
右上の 「＋」ボタン をクリック
2
メニューから 「New repository」 をクリック
▼ 「New repository」画面の入力例
Repository name ＊
yappi7000 /
← 英語・数字・ハイフンのみ！日本語NG

Create repository
🚨 絶対に守ること
・リポジトリ名は 英語・数字・ハイフンだけ（日本語NG）
・「Public」 を選ぶ（Privateだと公開できません）
・「Add README」をON にする（これをしないとアップロードできません）
✅ 作成成功の確認
「README.md」というファイルが表示されて、「Public」 バッジが付いていればOKです。
3
アップロード前にファイル名を変える
これが一番大事な準備です！
🚨 超重要！ここを間違えると公開されません
アップロードするHTMLファイルの名前を 必ず「index.html」 に変えてください。

❌ ダメな例：iphone_roadmap.html
❌ ダメな例：MNP完全ガイド.html（日本語NG）
✅ 正しい例：index.html
Mac
① Finderでファイルを探す
② ファイルを 1回クリック（選択状態にする）
③ 少し間をおいて もう1回クリック（ダブルクリックではない！）
④ 名前が青くなったら全部消して index.html と入力
⑤ Enterキー を押す
Win
① ファイルを 右クリック
② 「名前の変更」をクリック
③ 全部消して index.html と入力
④ Enterキー を押す
✅ 変更後の確認
ファイル名が index.html になっていればOK！
他の文字が入っていたらやり直してください。
4
index.html をアップロードする
ドラッグ＆ドロップするだけ
▼ リポジトリ画面：「Add file」ボタンの場所
main ▾
Go to file
Add file ▾ ← ここ！
Code
Create new file
← Upload files
1
「Add file ▼」 をクリック
2
「Upload files」 をクリック
3
画面中央のエリアに 「index.html」 をドラッグ＆ドロップ
（ファイルをつかんで、そのエリアの上で離すだけ）
4
ページを下にスクロールして 緑色の「Commit changes」ボタン をクリック
✅ アップロード成功の確認
ファイル一覧に 「README.md」と「index.html」の2つ が表示されていればOKです。
⚠️ ファイル名が index.html 以外になってしまったら
① ファイル名をクリック → ② 鉛筆アイコン（✏️）をクリック
→ ③ 上部のファイル名欄を 全部消して「index.html」と入力
→ ④ 「Commit changes...」→「Commit changes」

⚠️ 日本語が入ったファイル名はこの方法で変更できません！
その場合は：ファイルを削除（右上の「...」→「Delete file」）してから、
パソコンでファイル名を index.html に変えて再アップロードしてください。
5
GitHub Pagesを有効にする
URLを公開するための設定
▼ リポジトリ上部のメニュー：「Settings」の場所
Code
Issues
Actions
Settings ← ここ！
1
リポジトリ上部のメニューから 「Settings」 をクリック
2
左サイドバーを少し下にスクロールして 「Pages」 をクリック
3
「Branch」の欄に 「None ▼」 ボタンがあるのでクリック → 「main」 を選択
4
「Save」 ボタンをクリック
▼ Pages設定画面のイメージ
Build and deployment
Branch
None ▼ → main に変える！
/ (root)
Save ← クリック！
⏱ 1〜2分待ってください
「Save」を押してからページが公開されるまで 1〜2分 かかります。
すぐ開くと404エラーが出ることがありますが、少し待ってから再度アクセスしてみてください。
✅ 設定完了の確認
「Your GitHub Pages site is currently being built from the main branch.」
という文章が表示されれば設定完了です！
6
URLにアクセスして確認する
1〜2分待ってからアクセス
🌐 URLの形式
https://ユーザー名.github.io/リポジトリ名/

例：https://yappi7000.github.io/iphone-resale-MNP/
✅ うまくいかない時のチェックリスト
□ リポジトリが 「Public」 になっているか
□ ファイル名が 「index.html」（小文字・英語のみ）になっているか
□ Settings → Pages → Branchが 「main」 になっているか
□ Saveを押してから 2分以上 待ったか
□ ファイルが 真っ白 になる場合はファイルサイズが大きすぎる可能性あり
🎉
